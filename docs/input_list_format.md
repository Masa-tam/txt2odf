# 入力リストファイル フォーマットガイド

`txt2odf` では、変換したいテキストファイルと、見出し（章や節のタイトル）の構成を **「リストファイル」** と呼ばれる1つのテキストファイルにまとめて指定します。
このリストファイルを `-i` オプションでプログラムに渡すことで、指定された順番通りにファイルを結合し、見出しを挿入して1つの文書（ODT）を作り上げます。

---

## 1. 記述ルール

リストファイルは単なるテキストファイル（`.txt`）です。
各行の「先頭の文字」によって、その行が「見出し」なのか「読み込むファイル」なのかを判定します。

| 記号 | 意味 | 記述例 | ODTでの見た目 |
| :--- | :--- | :--- | :--- |
| `#` | 大見出し（章） | `# 第一章 出会い` | 最も大きな文字（テンプレートの `h1` サイズ）で表示されます。 |
| `##` | 中見出し（節） | `## 1. 運命の朝` | 少し大きめの文字（テンプレートの `h2` サイズ）で表示されます。 |
| `###` | 小見出し（項） | `### (1) 学校への道` | 本文より少し大きな文字（テンプレートの `h3` サイズ）で表示されます。 |
| `@` | テキストファイルの読み込み | `@chapter1/part1.txt` | 指定したテキストファイルの中身が、本文としてここに挿入されます。 |

*   **空白・空行について**: ファイル内の空行はすべて無視されます。見やすくするために自由に空行を入れて構いません。
*   **ファイルパスについて**: `@` で指定するファイルパスは、**リストファイル自体があるフォルダを基準**とした相対パス、もしくは絶対パスで指定します。
*   **エラー行**: 上記の記号（`#`, `##`, `###`, `@`）以外で始まる文字が書かれていた場合、構文エラーとして警告が出力され、その行は無視されます。

---

## 2. 記述例

例えば、`txt` フォルダの中に `list.txt` と各種テキストファイルが以下のように配置されているとします。

```text
txt/
 ├─ list.txt         ← このリストファイル
 ├─ prologue.txt
 └─ chapter1/
     ├─ part1.txt
     └─ part2.txt
```

このとき、`list.txt` の中身を次のように記述します。

```text
# プロローグ
@prologue.txt

# 第一章 夏の始まり
## 1. 憂鬱な登校
@chapter1/part1.txt

## 2. 予期せぬ出会い
@chapter1/part2.txt
```

### このリストファイルがどのように変換されるか

プログラムはこの `list.txt` を上から順番に読み込み、以下のように ODT を構築します。

1.  **「プロローグ」** という大見出し（`h1`）を作成。
2.  `prologue.txt` の中身を読み込んで本文として追加。
3.  **「第一章 夏の始まり」** という大見出し（`h1`）を作成。
4.  **「1. 憂鬱な登校」** という小見出し（`h2`）を作成。
5.  `chapter1/part1.txt` の中身を読み込んで本文として追加。
6.  **「2. 予期せぬ出会い」** という小見出し（`h2`）を作成。
7.  `chapter1/part2.txt` の中身を読み込んで本文として追加。

このように、複数のテキストファイルをまとめつつ、間に自由に見出しを挟み込んで一冊の本のような構造を簡単に作ることができます。
